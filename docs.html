Documentation

Paragraphs

Koara:

This is a simple paragraph.

Use the return key for lines
that need a break.

HTML:

<p>This is a simple paragraph.</p>

<p>Use the return key for lines<br>
that need a break.</p>

If enabled, any default blockelement will become a paragraph. Paragraphs are separated by one or more blank lines. Koara uses return-based line breaks.

Headings

Koara:

= Heading 1
== Heading 2
=== Heading 3
==== Heading 4
===== Heading 5
====== Heading 6
\= A heading can be escaped like this.

HTML:

<h1>Heading 1</h1>

<h2>Heading 2</h2>

<h3>Heading 3</h3>

<h4>Heading 4</h4>

<h5>Heading 5</h5>

<h6>Heading 6</h6>

<p>= A heading can be escaped like this.</p>

If enabled and a line starts with equal signs, it will be considered a heading. The maximum header-level is 6. Whitespace or blank lines are always optional.

Escaping: If you don't want koara to parse an equal-sign as a heading, you can escape it, by prepending a backslash. You can also escape all other characters used in the syntax.

Lists

Koara:

- Item 1
- Item 2
    - Item 2A
    - Item 2B
1. Item 1
2. Item 2
    1. Item 2A
    4. Item 2B

HTML:

<ul>
  <li>Item 1</li>
  <li>
    <p>Item 2</p>
    <ul>
      <li>Item 2A</li>
      <li>Item 2B</li>
    </ul>
  </li>
</ul>

<ol>
  <li>Item 1</li>
  <li>
    <p>Item 2</p>
    <ol>
      <li>Item 2A</li>
      <li value="4">Item 2B</li>
    </ol>
  </li>
</ol>

Unordered lists start with a dash before the item. Ordered lists start with a digit, followed by a dot. You can use any digit, koara will preserve the numbering when parsed.

To create sublist-items you have to use indentation. Default indentation is 4 spaces or a tab. List items can consist of multiple block elements. Because these contain blank lines, you have to use at least 2 blank lines to break out of the list or don't start the line with indentation.

Links

Koara:

This is an [internal link].
This is an [inline link](http://www.google.com).

HTML:

<p>This is an <a href="">internal link</a>.<br>
This is an <a href="http://www.google.com">inline link</a>.</p>

Koara supports internal and inline links. Internal links are useful for wiki and CMS applications. By default it returns an empty link, because the url should be resolved by the application.

Inline links are simply a combination of the link text and the url.

Images

Koara:

This is an [image:internal image]
This is an [image:inline image](http://www.koara.io/logo.png)

HTML:

<p>This is an <img src="" alt="internal image" /><br>
This is an <img src="http://www.koara.io/logo.png" alt="inline image" /></p>

Images work exactly the same as links, except they have an image: label.

Images and links are both resources. If you configure the parser to allow links but not images, the syntax will give you a natural fallback: Images will then become links with a prefix image: in the title.

Formatting

Koara:

Koara supports *strong* and _emphasis_

You can format *more then
one line*

HTML:

<p>Koara supports <strong>strong</strong> and <em>emphasis</em></p>

<p>You can format <strong>more then<br>
one line</strong></p>

If enabled, text between asterisks will be rendered as bold text, text between underscores will be rendered as italic text

Formatting can span over multiple lines. Intraword emphasis is allowed.

Blockquotes

Koara:

> Blockquote 1
>> Blockquote 2
>>> Blockquote 3

HTML:

<blockquote>
  <p>Blockquote 1</p>
  <blockquote>
    <p>Blockquote 2</p>
    <blockquote>
      <p>Blockquote 3</p>
    </blockquote>
  </blockquote>
</blockquote>

If enabled, you can use email-style blockquoting. Blockquotes can also be nested.

Code

Koara:

Inline `code` works with backticks. You can also create a fenced code block

```
function greeting() {
  console.log("Hello World");
}
```

HTML:

<p>Inline <code>code</code> works with backticks. You can also create a fenced code block</p>

<pre><code>function greeting() {
  console.log(&quot;Hello World&quot;);
}</code></pre>

If enabled, text between backticks will be rendered as code text. This can span over multiple lines.

Koara also support fenced code blocks. A fenced code block starts at the beginning of a line with at least 3 backticks. It can contain an optional 'language' identifier.
