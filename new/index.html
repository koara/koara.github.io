What is Koara?

When building an application, you can use a lightweight markup language to enhance the user's writing experience. But these languages are inflexible. You either allow all the syntax or nothing. This is not always appropriate, so most developers end up with a custom language or a fork of an existing language. It requires the user to each time learn a different language and jeopardizes portability between applications.

Koara is a modular lightweight markup language. The developer decides which parts of the syntax (modules) are allowed to be parsed. The rest will render as plain text. Koara comes with a formal grammar and a compliancy testsuite.

Modules

Paragarphs

If enabled, any default block element will become a paragraph. Paragraphs are separated by one or more blank lines. Koara uses return-based line breaks.

Headings

If enabled and a line starts with equal signs, it is considered a heading. The largest heading level is 6. Whitespace or blank lines are always optional in Koara.
If you don't want Koara to parse a line as a heading, escape the equal sign by prepending a backslash. You can also escape all other characters used in the syntax.

Lists

Unordered lists start with a dash before the item. Ordered lists start with a digit, followed by a dot. You can use any digit, Koara will preserve the numbering when parsed.

To create sublist-items you have to use indentation. Default indentation is 2 spaces or a tab. List items can consist of multiple block elements. Because these can contain blank lines, you have to use at least 2 blank lines to break out of the list or start a new block without indentation.

Links

Koara supports internal and inline links. Internal links are useful for wiki and CMS applications. By default it returns an empty link because the url should be resolved by the application.

Inline links are simply a combination of the link text and the url.

Images

Images work exactly the same as links except they have an image: label. Images and links are both resources. If you configure the parser to allow links but not images, the syntax will give you a natural fallback: Images will then become links with a prefix image: in the title.

Formatting

If enabled, text between asterisks will be rendered as bold text, text between underscores will be rendered as italic text. Formatting can span over multiple lines, and intraword emphasis is allowed.

Blockquotes

If enabled, you can use email-style blockquoting. Blockquotes can also be nested.

Code

If enabled, text between backticks will be rendered as code text. It can span over multiple lines.

Koara also support fenced code blocks. A fenced code block starts at the beginning of a line with at least 3 backticks. It can contain an optional 'language' identifier.
