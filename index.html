What is koara?

When building an application, you can use a lightweight markup language to enhance the user's writing experience. But these languages are inflexible. You either allow all the syntax or nothing. This is not always appropriate, so most developers end up with a custom language or fork of an existing language. It requires the user to each time learn a different language and jeopardizes portability between applications.

Koara is a modular lightweight markup language. The developer decides which parts of the syntax are allowed to be parsed. The rest will render as plain text.

How does it work?

Let's take the following sentence, written in koara: 
The quick brown [fox](http://getfirefox.com) jumps *over* the `lazy` dog
If I configure the parser to allow all the syntax, and we parse it to HTML, this will be the outcome: 
<p>The quick brown <a href="http://getfirefox.com">fox</a> jumps <strong>over</strong> the <code>lazy</code> dog</p>
If I configure the parser to allow only paragraphs and formatting, the output will look like this: 
<p>The quick brown [fox](http://getfirefox.com) jumps <strong>over</strong> the `lazy` dog</p>
If I configure the parser to allow only paragraphs, this will be rendered: 
<p>The quick brown [fox](http://getfirefox.com) jumps *over* the `lazy` dog</p>
