Lightweight markup languages are typically all-or-nothing, but most use cases require only a subset of the language. This leads to numerous forks and flavors and ultimately confuses the end-user. Koara is a modular lightweight markup language (based on a formal grammar) where every part of the syntax can be enabled/disabled. If the syntax is disabled, it will just render as plain text.

Paragraphs

When enabled, any block of text that is not a heading, list, blockquote or codeblock is considered a paragraph. Paragraphs are separated by one or more blank lines. Newline characters are translated to linebreaks. Whitespace before or after the text is trimmed.

This is a simple paragraph.

Use the return key for lines
that need a break.

Headings

A heading is a line that starts with 1 to 6 equal-signs. A space after the sign is not required. If a line should not be rendered as a heading, you can escape it by using a backslash. (you can also escape *, `, \, -, ., >, [, (, <, ], ), _.

= Heading 1
== Heading 2
=== Heading 3
==== Heading 4
===== Heading 5
====== Heading 6
\= A heading can be escaped like this.
